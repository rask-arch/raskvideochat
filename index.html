Suoni Notifiche

⚠️ Firestore: permessi mancanti

Le Security Rules di Firestore bloccano le scritture. Copia le regole qui sotto nel tuo progetto Firebase.

Vai su console.firebase.google.com → tuo progetto → Firestore Database → tab Rules → incolla questo e clicca Publish:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // RaskVideochat — allow any authenticated user (including anonymous)
    // to read and write all app data under the public namespace.
    match /artifacts/{appId}/public/data/{document=**} {
      allow read, write: if request.auth != null;
    }

    // Deny everything else
    match /{document=**} {
      allow read, write: if false;
    }
  }
}
💡 Alternativa rapida (solo per test/sviluppo):
Puoi anche usare queste regole temporanee ⚠️ NON usare in produzione:
allow read, write: if true;
📌 Anche Firebase Authentication: Per gli ospiti abilita Anonymous in Authentication → Sign-in method.
